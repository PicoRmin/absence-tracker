# نقشه راه بازسازی پروژه Story Image Generator

## هدف کلی
بازسازی پروژه به شکلی که:
- **کد تمیز و قابل توسعه** باشد
- **ساختار چندفایلی** داشته باشد (HTML، CSS، JS / data)
- **قابلیت‌های جدید و حرفه‌ای** اضافه شود
- **تجربه موبایل و PWA** بهبود یابد
- **خوانایی، نگهداری و مقیاس‌پذیری** بالا رود

## تحلیل اولیه وضعیت فعلی
- تمام پروژه در یک فایل HTML نگه‌داری شده است.
- UI و منطق و داده‌ها با هم ترکیب شده‌اند.
- رندر Canvas با توابع داخلی انجام می‌شود، اما کد زیادی تکراری و بدون جداسازی است.
- قابلیت‌های پایه خوب است، اما:
  - فرم‌ها به صورت `innerHTML` ساخته شده‌اند.
  - state management ساده و نامنسجم است.
  - پشتیبانی از ویژگی‌های جدید دشوار خواهد شد.

## پیشنهاد ساختار پروژه
1. `index.html`  
   - قالب HTML پایه
   - فقط ساختار صفحه و placeholderها
2. `styles.css`  
   - تمام CSS جدا و با متغیرهای CSS
   - سبک‌های ریسپانسیو و حالت موبایل
3. `data.js`  
   - داده‌های ثابت: تم‌ها، دسته‌ها، قالب‌ها
4. `app.js`  
   - state management
   - رندرینگ Canvas و UI
   - منطق رویدادها
5. `utils.js`  
   - توابع کمکی مانند roundRect، wrapText، export
6. `README.md`  
   - مستندات پروژه
7. `roadmap.md`  
   - این فایل نقشه راه

## پیشنهادهای بهبود خلاقانه و خلافانه
- **پشتیبانی PWA**: نصب روی موبایل و استفاده آفلاین
- **ذخیرهٔ تنظیمات کاربر**: localStorage برای بازگشت به آخرین قالب و رنگ
- **قابلیت آپلود عکس پس‌زمینه یا لوگو**: ترکیب متن با تصویر سفارشی
- **افزودن حالت شب/روز خودکار** و انتخاب کاربر
- **تنظیمات پیشرفته‌تر**: کنترل تایپوگرافی، فاصله خطوط، padding
- **جستجو و فیلتر قالب‌ها**: پیدا کردن قالب با نام یا گروه
- **قابلیت export SVG** علاوه بر PNG
- **قابلیت Undo/Redo ساده** برای تغییرات سریع
- **اضافه کردن انیمیشن سبک** هنگام تغییر قالب یا تم
- **حالت گرید قالب‌ها با پیش‌نمایش پویا** برای انتخاب بهتر

## فازهای اجرایی

### فاز 1: جداسازی و پایه
- ایجاد `index.html` با ساختار اولیه
- جدا کردن CSS به `styles.css`
- جدا کردن داده‌ها به `data.js`
- جداسازی منطق به `app.js`
- اطمینان از کارکرد مشابه با نسخه فعلی

### فاز 2: بازسازی UI و بهینه‌سازی
- بازنویسی بخش‌های sidebar، gallery، customize
- تبدیل `innerHTML` های پیچیده به DOM API یا template literals ایمن
- بهبود ریسپانسیو و ناوبری موبایلی
- افزودن حالت “mobile bottom sheet” استانداردتر

### فاز 3: امکانات جدید
- افزودن ذخیرهٔ localStorage برای state
- افزودن export SVG و download custom filename
- افزودن جستجوی قالب و فیلتر دسته
- افزودن تنظیمات فونت و تایپوگرافی

### فاز 4: PWA و آفلاین
- اضافه کردن `manifest.json`
- اضافه کردن `service-worker.js`
- کش استایل‌ها و اسکریپت‌ها
- افزودن اعلان نصب روی موبایل

### فاز 5: بهینه‌سازی و کیفیت
- اصلاح خطاهای دستوری و ساختاری
- بهینه‌سازی بارگذاری فونت و تصاویر
- اضافه کردن مستندات کاربردی در `README.md`
- در صورت نیاز، افزودن ESLint / Prettier (با فایل ساده)

## مواردی که بعداً می‌توان اضافه کرد
- **پشتیبانی از فونت‌های کاربر** و آپلود فونت
- **اشتراک‌گذاری مستقیم روی شبکه‌های اجتماعی**
- **ساخت قالب از روی عکس کاربر** (AI prompt یا Layer)
- **پشتیبانی چند زبانه** (فارسی + انگلیسی)
- **ساخت گالری قالب‌های متحرک**

## گام بعدی
- مرحله اول: ساختار چندفایلی را بسازیم و نسخه تک‌فایلی را به اجزا تقسیم کنیم.
- بعد از آن: سیستم مدیریت state و رندر Canvas را بازسازی می‌کنیم.
- سپس: قابلیت‌های جدید و PWA را به پروژه اضافه می‌کنیم.
